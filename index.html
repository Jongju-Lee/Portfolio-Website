Slider 등 꼭 필요한 기능성 플러그인을 제외하고 네비게이션, 탭 메뉴, 어코디언 등 상호작용을 위한 컨텐츠에는 플러그인을 사용하지 않고 JavaScript, jQuery를 병행해서 활용하며 직접 하드코딩 해서 작업합니다.

M

Mobile Web App

Mobile Web App을 제작한 경험이 있습니다. 더 나은 UI/UX를 위해 디자인 뿐만 아니라 효율적인 컨텐츠 배치를 우선적으로 고민하여 제작합니다. 반응형 웹, 적응형 웹, 모바일 웹앱, 하이브리드 앱, 네이티브 앱의 차이점을 이해하고 있습니다.

RW

Responsive Web

반응형 웹은 안정적인 기존방식인 position, float, media query 등을 활용한 반응형 웹 제작 경험이 충분하고, 좀 더 반응형에 특화된 레이아웃이 필요한 경우 Grid, Flex를 사용한 방식으로도 작업이 가능합니다.

FR

CSS Framework

Bootstrap, MUI, Ant Design, UIKit등 여러가지 Framework를 사용해본 경험이 있고 충분한 이해도를 가지고 있습니다. 그 중, 크로스브라우징이 잘 되어있고 CSS Custom이 비교적 편한 UIKit을 메인으로 사용하고 있습니다.

SC

SCSS, CSS Nesting

CSS를 보다 더 효율적으로 작성하고 유지보수성을 높이기 위해서 SCSS를 사용합니다. SCSS는 작업시 매번 컴파일을 해줘야 한다는 단점이 있지만 컴파일이 필요없는 최신 문법인 CSS Nesting도 사용 가능합니다.

Fig

Epilogue.

감사합니다!
웹 퍼블리셔 이 종 주 였습니다.

아직 완벽한 역량을 갖추진 못했습니다. 하지만 정말 잘하고 싶고, 제가 잘 할수 있는 일이라는 확신이 들었기 때문에 지금껏 오랜 시간동안 해왔던 일을 마무리 짓고 새 출발을 하려고 합니다.
항상 겸손한 마음가짐으로 배움을 게을리 하지 않으며 꾸준히 쌓아온 탄탄한 기본기와 빠른 습득력을 토대로 트렌드에 뒤쳐지지 않는 웹 퍼블리셔가 되겠습니다.

퍼블리싱 원칙과 기본기

원활한 작업을 위해 폴더구조와 경로를 체계적으로 정리하고, 항상 깔끔한 인덴트와 적절한 주석 사용으로 저 뿐만 아니라 협업자도 알아보는데 불편함이 없도록 깔끔하고 가독성이 좋은 코드작성을 선호합니다.

하드코딩

디자인이 아닌 기능적으로 필요한 경우에만 플러그인을 사용하고 그 외 모든 작업은 하드코딩을 원칙으로 작업합니다. Position, Float 속성을 사용하는 기존방식과, Flex, Grid를 사용한 레이아웃 모두 가능하고, 반복 또는 불필요한 코드는 최대한 압축하여 코드를 작성합니다.

체계적인 퍼블리싱

작업에 들어가기 전 벤치마킹 사이트를 물색하고 와이어프레임을 작성한 후, UI 디자인, 기능정의서, HTML구조 설계 순으로 체계적인 퍼블리싱을 합니다. 순차적인 단계를 거침으로써 프로젝트 중 발생하는 불필요한 과정을 줄이고 효율적으로 퍼블리싱을 하고 있습니다.

항상 배우려는 자세

새로운 기술을 학습하는것 에 대한 욕심이 있고 배우고 사용하는 과정에서 느끼는 보람과 성취감을 좋아합니다. 현재에 안주하지 않고 항상 배움을 실천하는 퍼블리셔가 되려고 노력하고 있습니다.
