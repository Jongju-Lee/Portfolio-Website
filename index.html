Publishing Skill

HTML5

전자 교과서 퍼블리싱 프로젝트에서 HTML5의 Semantic Tag를 적극 활용하여 검색 엔진 최적화(SEO)를 고려한 설계를 수행하였습니다. 웹 표준 및 웹 접근성, 반응형 UI를 고려하여 HTML 구조를 먼저 설계한 후 Markup을 진행합니다.

CSS3

CSS는 BEM 방법론을 적용하여 일관된 Class naming convention을 유지합니다. 공통 UI 설계와 재사용 가능한 스타일 코드 작성으로 프로젝트 효율성을 높였으며, Flexbox와 Grid 및 MediaQuery를 활용한 반응형 레이아웃 경험이 많습니다. 큰 규모의 프로젝트의 경우 SCSS 또는 CSS Nesting를 활용해 불필요한 선택자 작성을 최소화하고 가독성을 높여 생산성 및 유지 보수성을 높일 수 있습니다.

JS, jQuery

JavaScript를 활용한 DOM 조작 및 이벤트 제어에 익숙하고 Interaction 기능을 직접 구현할 수 있습니다. 상태 관리 및 조건 분기 로직 등을 통해 사용자 중심의 동적인 UI를 제작한 경험이 있습니다. 전자 저작물 등의 Legacy 프로젝트에서는 Cross-Browser 호환성 및 생산성을 위해 jQuery를 주로 활용하였습니다.

Web Accessibility

웹 접근성(WCAG) 기준을 준수하며 HTML Semantic Tag 구조 설계와 함께 tabindex, aria 속성 등 접근성 관련 Attr을 적극 활용해 실제 시각장애인 사용자와 스크린 리더를 활용한 테스트 를 거쳐 전자 저작물의 접근성 평가 기준 100% 충족한 실전 경험을 보유한 실전 경험이 있습니다.

Responsive UI, Cross-Browser

Media Query, Flexbox 및 Grid를 활용하여 다양한 해상도 및 디바이스 대응이 가능합니다. Legacy Browser 기반 런처 환경 에서도 일관된 UI/UX를 제공하기 위해 reset.css를 사용, 호환성을 고려한 CSS 코드 작성, CSS Fallback을 구현하여 다수의 프로젝트를 수행한 경험이 있습니다.

React, CSS Module, CSS-in-JS

React를 활용하여 components 기반의 Modern UI를 구축한 경험이 있습니다. CSS Module화를 통해 스타일 충돌 방지 및, 유지 보수성을 높이고, 동적 스타일링이 필요한 프로젝트에서는 Styled-Components를 활용하여 구현 가능합니다.

Git, SVN

Git을 활용한 버전 관리 및 branch 운영, 코드 충돌 최소화, history 관리 경험이 있고 Legacy Browser 기반 전자 저작물 프로젝트에서는 SVN을 활용한 경험이 있어 형상 관리에 능숙합니다.

Figma

퍼블리싱 원칙과 기본기

원활한 작업을 위해 폴더구조와 경로를 체계적으로 정리하고, 항상 깔끔한 인덴트와 적절한 주석 사용으로 저 뿐만 아니라 협업자도 알아보는데 불편함이 없도록 깔끔하고 가독성이 좋은 코드작성을 선호합니다.

하드코딩

디자인이 아닌 기능적으로 필요한 경우에만 플러그인을 사용하고 그 외 모든 작업은 하드코딩을 원칙으로 작업합니다. Position, Float 속성을 사용하는 기존방식과, Flex, Grid를 사용한 레이아웃 모두 가능하고, 반복 또는 불필요한 코드는 최대한 압축하여 코드를 작성합니다.

체계적인 퍼블리싱

작업에 들어가기 전 벤치마킹 사이트를 물색하고 와이어프레임을 작성한 후, UI 디자인, 기능정의서, HTML구조 설계 순으로 체계적인 퍼블리싱을 합니다. 순차적인 단계를 거침으로써 프로젝트 중 발생하는 불필요한 과정을 줄이고 효율적으로 퍼블리싱을 하고 있습니다.
